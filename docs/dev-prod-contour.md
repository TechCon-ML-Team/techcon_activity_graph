# Dev/Prod контур — techcon_activity_graph

> Краткая карта роли репозитория в контуре разработки и развёртывания TechCon.
> Источник истины по фактическому состоянию runtime/деплоя — инженерная проверка + `dev/status.md`.

## Роль репозитория

**Вспомогательный сервис визуализации** (форк `Ashutosh00710/github-readme-activity-graph`),
рендерит график активности GitHub для поверхностей TechCon. Это **НЕ основной сервисный/продуктовый
репозиторий** платформы — утилита/портфель с собственным runtime, но вне центральной продуктовой
deploy-матрицы.

- Runtime: Node.js + TypeScript (Express, вход `src/main.ts`).
- Целевое развёртывание: **Vercel** (`vercel.json`, serverless `@vercel/node`). Деплой выполняется
  внешней интеграцией Vercel (не workflow'ами в репозитории), а не данным контуром.
- Режим сопровождения: низкая изменчивость, правки только по подтверждённой необходимости.

## Состояние веток (на момент аудита)

- **Default-ветка: `main`** — корректно (переименование не требуется).
- `origin/main` — единственная рабочая/релизная ветка.
- Унаследованные от upstream-форка feature-ветки (`add-border-color-option`, `dates-feature`,
  `feature/custom-timeframe`, `fix-diagonal-tilt-issue`, `wesite-changes-for-theme-selection`),
  а также `development` и `backup` — **застойные**, к контуру TechCon не относятся.

## CI

`.github/workflows/node.js.yml`:
- `push`/`pull_request` в `main` + `workflow_dispatch` → `npm ci && npm test` (Node 22).
- При успехе на `main` job `notify-hub` отправляет `repository_dispatch`
  (`knowledge-sync-requested`) в `TechCon-ML-Team/techcon_hub`.

## Рекомендуемая модель контура

Для вспомогательного репозитория с одним внешним деплоем (Vercel) избыточна схема
`develop→dev / main→prod`. Рекомендуется **single-main / manual**:

- **`main` = единственная долгоживущая ветка.** Изменения через короткоживущие topic-ветки
  (`<type>/<slug>`) и PR в `main`.
- **Default-ветка остаётся `main`** (изменений не требуется).
- **Промоушн в прод** (Vercel) — по merge в `main` (или вручную через Vercel), осознанно, не на
  каждый коммит-черновик.
- Прибрать застойные ветки (`development`, `backup`, наследие форка) — отдельным решением владельца,
  чтобы состояние веток отражало реальный контур.

## Branch-protection (рекомендация, применяет админ централизованно)

- `main`: required PR review, запрет прямых пушей, обязательный зелёный статус `Node.js CI / build`.
- Линейная история / squash-merge для topic-веток.
- Эти настройки — зона админ-централизованного управления; данным контуром не применяются.

## Границы данной работы

- Default-ветка, branch-protection, прод-деплой, прод-рантайм — **не трогались** (админ-зона).
- Зафиксированы только рекомендации и роль репозитория в контуре.
